## 使用方法

1. 根据创作需求，从目录中选择合适框架
2. 读取 `frames/<编号>-<名称>/` 目录下的文件：
   - `README.md`：框架原理、结构、适用场景、注意事项
   - `prompt.md`：可直接使用的 Prompt 模板（含配套技法）
   - `examples/`：市场验证过的实际案例
3. 将 Prompt 模板中的占位符替换为用户的实际内容
4. 参考 `examples/` 中的案例，了解框架的实际应用

## 核心配套技法（通用）

以下技法可搭配任意框架使用，详见各框架文档：

### 技法一：抽象接具象
- **定义**：抽象观点 + 具体例子，信息密度拉满
- **示例**：「引力是信息熵减小的表现。比如星云凝聚成星球，不就是无数个1合并成少数1的过程吗」

### 技法二：引用学说/名言
- **定义**：用权威理论背书，普通观点升级为洞察
- **示例**：「心理学上的「自我损耗理论」指出，每一次自我控制都会消耗有限的意志力资源」

### 技法三：观点陌生熟悉化
- **定义**：新词命名 / 蹭热度 / 反常识颠覆，制造新鲜感+共鸣感
- **三种实现方式**：
  1. **创造/借用新词**：斩杀线、情绪劳动、认知过载
  2. **蹭热度**：借势热点/热梗/热词
  3. **反常识颠覆**：颠覆固有认识，提出更接近真相的结论

## 质量要求（来自 MEMORY.md）

1. **风格偏好**：
   - 实战导向，深度分析
   - 直接行动，避免套话
   - 例子具体可感，避免常见信息
   - 学术引用要深入、准确

2. **信息密度原则**：
   - 陌生感 ↔ 熟悉感交替
   - 每个信息点都要有增量
   - 节奏快，不停留

3. **结构要求**：
   - 每个框架必须包含：理论说明 + Prompt模板 + 实际案例
   - 案例必须经过市场验证
   - 标注技法使用，便于学习和复制
